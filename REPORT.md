# Отчет

## Что было сделано

Есть два основных класса:

- `StringGenerator` - создает тестовые массивы строк;
- `StringSortTester` - запускает сортировки, измеряет время и записывает результаты.

Строки генерируются из латинских букв, цифр и специальных символов. Длина строк - от 10 до 200 символов. Размеры массивов меняются от 100 до 3000 строк с шагом 100.

## Какие данные тестировались

Использовались четыре типа массивов:

- случайные строки
- строки в обратном порядке
- почти отсортированные строки
- строки с общими префиксами

## Какие сортировки сравнивались

Были реализованы и протестированы:

- обычный `QuickSort`
- обычный `MergeSort`
- тернарный `String QuickSort`
- `String MergeSort` с использованием LCP
- `MSD Radix Sort`
- `MSD Radix Sort` с переключением на `String QuickSort` для маленьких подмассивв

## Что измерялось

Для каждого алгоритма измерялись:

- среднее время работы
- количество посимвольных сравнений или просмотров символов
- корректность результата сортировки

Каждый тест запускался несколько раз, после этого считалось среднее значение

## Как запустить программу

```bash
g++ -std=c++20 -O2 -Wall -Wextra -pedantic main.cpp -o string_sort_bench
./string_sort_bench 3000 100 5 string_sort_results.csv
```

После запуска появляется файл `string_sort_results.csv` с результатами.

## Как построить графики

Для построения графиков используется `matplotlib`

```bash
python3 plot_results.py string_sort_results.csv plots png
```

После этого графики сохраняются в папку `plots`

## Вывод

Обычные сортировки хорошо работают на случайных строках, но хуже показывают себя на строках с длинными общими префиксами, потому что им приходится много раз сравнивать одни и те же символы.

Специализированные строковые сортировки лучше используют структуру строк. Особенно это заметно на данных с общими префиксами: количество посимвольных сравнений становится меньше.

`MSD Radix Sort` и тернарный `String QuickSort` обычно дают хорошие результаты, потому что они работают не с целыми строками сразу, а с отдельными символами. `String MergeSort` с LCP тоже уменьшает число сравнений, но в моей реализации по времени он может быть медленнее из-за дополнительных операций с массивами.

Все сортировки в тестах дали правильный результат.
